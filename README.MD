**I said no.**

`ferris-scan` is a **reaction against software decay**. Written in Rust, it delivers:

- âš¡ **Fearless Concurrency:** Multi-threaded scanning saturates your NVMe drive's true potential
- ğŸª¶ **Zero Bloat:** The entire binary is <3MB. Your browser tab uses more memory
- ğŸ”’ **Memory Safety:** No segfaults. No buffer overflows. No "Access Violation" crashes
- ğŸ¯ **Instant Feedback:** Real-time progress. Responsive TUI. Native performance

This is what happens when you rebuild a legacy tool *the right way*.
